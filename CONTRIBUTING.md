# 🤝 Contributing to Kurra

Thanks for your interest in contributing!

Before contributing, please read `VISION.md`.

Kurra is a product with a clear direction — not all contributions will be accepted.

---

## 🛠️ How to contribute

1. Fork the repository
2. Create a new branch
3. Make your changes
4. Open a pull request with a clear explanation

---

## ✅ What we are looking for

- Bug fixes
- Performance improvements
- UI/UX improvements (keeping simplicity)
- Improvements to lesson flow
- Better recall-based interactions

---

## ❌ What we are NOT looking for

- Large feature additions without discussion
- Complex systems that increase friction
- Features copied directly from other language apps without adaptation
- Anything that shifts Kurra away from a speak-first approach

---

## 💡 Proposing new features

Before building, open an issue with:

- Problem you are solving
- Proposed solution
- Why it aligns with Kurra’s vision

---

## 🧑‍💻 Code guidelines

- Keep code simple and readable
- Avoid over-engineering
- Prioritise clarity over cleverness

---

## 🎯 Final note

Kurra is intentionally opinionated.

We value product direction over feature quantity.
